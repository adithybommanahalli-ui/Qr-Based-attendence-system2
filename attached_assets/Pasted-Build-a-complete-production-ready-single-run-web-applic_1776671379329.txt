========================================
API ROUTES (ALL MUST BE IMPLEMENTED AND WORKING)
========================================
Auth:
- POST /api/auth/register  { name, email, password, role, branch?, section_code?, face_embedding? } → creates User; if role=student, also creates Student with face_embedding (JSON array) and links to section by code.
- POST /api/auth/login     { email, password } → returns { token, user }
- GET  /api/auth/me        (JWT) → current user

Sections (lecturer only):
- POST /api/section/create        { name } → auto-generates unique 6-char code
- GET  /api/section/mine          → list lecturer's sections with student counts
- GET  /api/section/<id>/students → list students in section

Sessions (lecturer only):
- POST /api/session/start  { section_id, duration_minutes, latitude, longitude, radius_meters } → creates session with qr_token (uuid4), start_time=now, end_time=now+duration, status=active. Returns session + QR image (base64 PNG) encoding the qr_token.
- POST /api/session/end    { session_id } → sets status=ended
- GET  /api/session/<id>   → session details + attendance list (live polling)
- GET  /api/session/active/<section_id> → current active session

Attendance (student only):
- POST /api/attendance/mark { qr_token, face_embedding (array of 128 floats), latitude, longitude }
  Validation order (reject with clear reason if any fails):
    1. Session exists and status=active and now < end_time (else auto-end it)
    2. Student belongs to session's section
    3. Not already marked (UniqueConstraint)
    4. Geolocation within radius (Haversine). Reject if > radius_meters.
    5. Face match: Euclidean distance between submitted embedding and stored embedding < 0.6 (threshold). Use numpy.linalg.norm.
  On success: insert Attendance(status='present'). On failure: insert Attendance(status='rejected', reason=...) ONLY if session+student valid, otherwise just return error.
- GET  /api/attendance/mine → student's attendance history

Use a `@token_required` decorator that parses `Authorization: Bearer <token>`, loads user into `request.current_user`. Add `@role_required('lecturer')` / `@role_required('student')`.

Return JSON `{ success: bool, data|error, message }` consistently. Wrap every route in try/except returning 400/401/403/404/500 with clear messages.
